TFG

Se trata de una SPA (Single Aplication Web) donde los usuarios pueden subir sus instrumentos y contactar con sus clientes para alquilarlos a partir de un chat. La web fue diseñada como una API REST, separando la programación del lado de cliente con la del lado del servidor, enviando los datos en formato JSON a través del protocolo HTTP. Debido a la forma de diseñar la aplicación elegida, se hizo uso de dos frameworks: Laravel en el lado del servidor y React en el lado del cliente. El motor de bases de datos utilizados es MariaDB.

Los enlaces a git están divididos en dos: la parte del back (Laravel) y la del front (React). Dado que el proyecto está desplegado en docker, también hay un enlace a dockerhub.

Click Competition

Se trata de una API REST que consiste en ir acumulando clicks sobre rectángulos que cambian de lugar cada x número determinado de clicks. Para el front se usó JavaScript con peticiones AJAX y para el back Spring Boot + Hibernate + H2 Database.

En la aplicación se usa tokens para autenticar a los usuarios.

CRUD

Aplicación hecha en PHP para gestiorar el CRUD (Creación, consulta, modificación y borrado) de una tabla incluyendo login/logout de usuarios para poder acceder. La aplicación consistirá en una página de login que debe comprobar nombre de usuario y contraseña (almacenados en una tabla creada al inicializar la base de datos), una principal donde aparecerá el listado de las filas almacenadas en la tabla y otra con el formulario para dar de alta filas o editarlas.

Lista de tareas

Block de notas realizado en React JS + Firebase donde el usuario puede crear, editar y borrar cualquier nota que escriba. Para ello, antes tienes que loguearte o registrarte.

Instagram

Este proyecto se trata de una aplicación web que intentará imitar a Instagram. Se podrá crear usuarios, subir fotos y comentarlas. Realizada en Laravel siguiendo el modelo MVC (Modelo, Vista, Controlador). La base de datos será MariaDB. EL proyecto está desplegado en Docker.
